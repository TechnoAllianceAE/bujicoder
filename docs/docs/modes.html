Cost Modes

Control the speed-quality-cost tradeoff for every task.

Overview

BujiCoder has four modes that determine which models are used and how agents behave:

ModePurposeSpeedCost
NormalEveryday coding tasksFastLow
HeavyComplex tasks requiring deeper reasoningMediumMedium
MaxMission-critical work, parallel evolutionSlowHigh
PlanRead-only analysis and documentationFastLow

Switching Modes

Switch modes in the TUI using any of these methods:

  • /mode normal, /mode heavy, /mode max, /mode plan
  • /mode (opens a visual picker)
  • Shift+Tab (cycles through modes)

Your selected mode is saved to the config file and persists across sessions.

Model Roles

Each cost mode assigns models to three roles:

RoleUsed byRecommendation
mainOrchestrator agent (routes tasks, drives conversation)Best model you can afford
file_explorerFile navigation, codebase searchFast, cheap model
sub_agentEditor, planner, researcher, reviewer, etc.Balance of quality and cost

Configuring Models per Mode

modes:
  normal:
    main: x-ai/grok-code-fast-1
    file_explorer: openai/gpt-4o-mini
    sub_agent: z-ai/glm-5
  heavy:
    main: qwen/qwen3.5-35b-a3b
    file_explorer: openai/gpt-4o-mini
    sub_agent: z-ai/glm-5
  max:
    main: minimax/minimax-m2.5
    file_explorer: openai/gpt-4o-mini
    sub_agent: z-ai/glm-5

Per-Agent Overrides

Within any cost mode, you can override the model for a specific agent using agent_overrides:

modes:
  max:
    main: anthropic/claude-sonnet-4
    file_explorer: openai/gpt-4o-mini
    sub_agent: anthropic/claude-sonnet-4
    agent_overrides:
      editor: anthropic/claude-sonnet-4
      planner: qwen/qwen3-235b-a22b
      researcher: google/gemini-2.5-pro-preview

Any agent not listed in agent_overrides uses the default sub_agent model. The orchestrator always uses main.

Plan Mode

Plan mode is a special read-only mode where BujiCoder:

  • Reads and analyses code for understanding
  • Creates and modifies .md documentation files
  • Does not modify source code

This is useful for project onboarding, architecture documentation, and code review analysis.

Max Mode — Parallel Evolution

In Max mode, when editing files, BujiCoder can spawn multiple agents to implement competing solutions in parallel. A judge agent then evaluates the results and picks the best one.

This is like running a coding tournament for every task — more expensive, but produces higher quality results.
