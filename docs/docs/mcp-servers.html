MCP Servers

Extend BujiCoder with any Model Context Protocol server.

What is MCP?

MCP (Model Context Protocol) is an open standard for connecting AI tools to external systems. MCP servers expose tools (functions) that BujiCoder can call — things like GitHub operations, browser automation, database queries, and more.

Adding MCP Servers

Via Config File

Add servers to the mcp_servers section in ~/.bujicoder/bujicoder.yaml:

mcp_servers:
  - name: filesystem
    command: npx
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/projects"]
    lazy: true

  - name: github
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]
    lazy: true

  - name: puppeteer
    command: npx
    args: ["-y", "@anthropic-ai/mcp-server-puppeteer"]
    lazy: true

Via the /mcp Command

Add or remove servers without leaving the TUI:

# List all configured servers and their status
/mcp

# Add a new server
/mcp add github npx -y @modelcontextprotocol/server-github

# Remove a server
/mcp remove github

Servers added via /mcp add are saved to the config file and default to lazy loading.

Server Config Fields

FieldTypeDescription
namestringUnique identifier for the server
commandstringThe command to launch the server process
argslistArguments passed to the command
lazyboolIf true, only start when a tool from this server is first invoked

Eager vs Lazy Loading

Eager (lazy: false)Lazy (lazy: true)
When startedOn BujiCoder launchOn first tool call
Startup impactSlower startupNo impact
First callInstantSmall delay while server starts
Best forFrequently used serversOccasionally used servers
Recommendation: Use lazy: true for most servers. This keeps startup fast and only pays the cost when you actually need the tools.

Example: GitHub Server with Token

Some servers need environment variables. Set them in your shell before launching BujiCoder:

# Set the token in your shell profile
export GITHUB_TOKEN="ghp_abc123..."

# Then add the server to config
mcp_servers:
  - name: github
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]
    lazy: true

Checking Server Status

Use /mcp in the TUI to see all configured servers, their status (running / lazy / stopped), and the tools they provide.

Troubleshooting

  • Server fails to start: Make sure npx is installed and the package name is correct. Try running the command manually in your terminal to see error output.
  • Tools not showing: Restart BujiCoder after adding servers to the config file, or use /refresh.
  • Permission errors: Some servers need API tokens or credentials passed via environment variables.
