Configuration

BujiCoder uses a single YAML config file for all settings.

Config File Location

The config file lives at:

~/.bujicoder/bujicoder.yaml

This file is created automatically by the setup wizard on first run. You can edit it manually at any time.

Override: Set the BUJICODER_CONFIG_DIR environment variable to use a different config directory.
One key is enough: You only need a single API key to get started. If you use OpenRouter (recommended), the default model mappings work out of the box — no modes section needed. If you use a different provider, you must set all models to ones available from that provider. See API Keys for per-provider sample configs.

Minimal Config (OpenRouter)

The simplest config — just add your OpenRouter key and go. The built-in defaults handle everything else:

# ~/.bujicoder/bujicoder.yaml
mode: local
cost_mode: normal

api_keys:
  openrouter: sk-or-v1-your-key-here

Full Config Example

Here's a complete config file showing all available options. This example uses OpenRouter, so all model IDs are in OpenRouter format (e.g. x-ai/grok-code-fast-1):

# BujiCoder Configuration
# You only need ONE provider key. All models below must be
# available from the provider(s) whose keys you configure.

mode: local
cost_mode: normal

api_keys:
  openrouter: sk-or-v1-abc123...       # OpenRouter — one key for 200+ models

modes:
  normal:
    main: x-ai/grok-code-fast-1
    file_explorer: openai/gpt-4o-mini
    sub_agent: z-ai/glm-5
  heavy:
    main: qwen/qwen3.5-35b-a3b
    file_explorer: openai/gpt-4o-mini
    sub_agent: z-ai/glm-5
  max:
    main: minimax/minimax-m2.5
    file_explorer: openai/gpt-4o-mini
    sub_agent: z-ai/glm-5
    agent_overrides:
      editor: anthropic/claude-sonnet-4
      planner: qwen/qwen3-235b-a22b

agents_dir: ~/.bujicoder/agents        # Custom agent definitions

mcp_servers:
  - name: filesystem
    command: npx
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/home/user"]
    lazy: true
  - name: github
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]
    lazy: true

Agent & Sub-Agent Model Mapping

Each cost mode assigns models to three roles: main (orchestrator), file_explorer (lightweight navigation), and sub_agent (all other specialist agents). You can override the model for individual agents using agent_overrides.

Provider match: All model IDs below are OpenRouter models. If you use a different provider (e.g. Anthropic-only), replace every model ID with one available from your provider. See per-provider sample configs. 
# Full agent model mapping example (OpenRouter)
mode: local
cost_mode: normal

api_keys:
  openrouter: sk-or-v1-your-key-here

modes:
  normal:
    main: qwen/qwen3.5-122b-a10b
    file_explorer: openai/gpt-oss-20b
    sub_agent: together/deepseek-ai/DeepSeek-V3.1
    agent_overrides:
      editor: deepseek/deepseek-v3.2-speciale
      git_committer: openai/gpt-oss-120b
      planner: z-ai/glm-5
      reviewer: together/moonshotai/Kimi-K2.5
      thinker: together/meta-llama/Llama-3.3-70B-Instruct-Turbo

  heavy:
    main: x-ai/grok-code-fast-1
    file_explorer: openai/gpt-oss-20b
    sub_agent: z-ai/glm-5
    agent_overrides:
      editor: together/Qwen/Qwen3-Coder-480B-A35B-Instruct-FP8
      git_committer: openai/gpt-oss-120b
      reviewer: moonshotai/kimi-k2.5
      thinker: together/deepseek-ai/DeepSeek-R1

  max:
    main: minimax/minimax-m2.5
    file_explorer: openai/gpt-oss-20b
    sub_agent: together/MiniMaxAI/MiniMax-M2.5
    agent_overrides:
      editor: together/Qwen/Qwen3-Coder-480B-A35B-Instruct-FP8
      git_committer: openai/gpt-oss-120b
      implementor: together/MiniMaxAI/MiniMax-M2.5
      judge: minimax/minimax-m2.5
      parallel_editor: minimax/minimax-m2.5
      planner: together/Qwen/Qwen3.5-397B-A17B
      researcher: together/MiniMaxAI/MiniMax-M2.5
      reviewer: qwen/qwen3.5-122b-a10b
      thinker: together/Qwen/Qwen3-235B-A22B-Thinking-2507
How resolution works: When an agent runs, BujiCoder checks agent_overrides first, then falls back to the role-based model (main, file_explorer, or sub_agent). This lets you assign expensive, high-quality models to critical agents (like editor) while keeping cheaper models for others.

Config Fields Reference

FieldTypeDescription
modestringAlways local
cost_modestringDefault cost mode: normal, heavy, or max
api_keysobjectAPI keys for LLM providers (see API Keys)
modesobjectModel assignments per cost mode (see Cost Modes)
agents_dirstringPath to custom agent YAML files (optional)
mcp_serverslistMCP server configs (see MCP Servers)

Environment Variables

These environment variables override or supplement the config file:

VariableDescription
BUJICODER_CONFIG_DIROverride config directory (default: ~/.bujicoder)
BUJICODER_AGENTS_DIROverride custom agents directory
BUJICODER_ENABLE_MOUSEEnable mouse capture in TUI
BUJICODER_DISABLE_UPDATE_CHECKSkip update check on launch
OPENROUTER_API_KEYOpenRouter API key (fallback if not in config)
ANTHROPIC_API_KEYAnthropic API key (fallback)
OPENAI_API_KEYOpenAI API key (fallback)

Project-Level Config

Create a .bujicoderrc file in your project root to set per-project permissions:

permissions:
  allowed_commands:
    - "go test"
    - "npm run"
    - "make"
  auto_approve_reads: true
