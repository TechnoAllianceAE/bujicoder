API Keys

BujiCoder talks directly to LLM providers using your API keys. No middleman, no markup.

Quick start: You only need one API key to get started. OpenRouter is the easiest since it gives access to 200+ models with a single key.

Adding Keys to the Config

Add your API keys under the api_keys section in ~/.bujicoder/bujicoder.yaml:

api_keys:
  openrouter: sk-or-v1-abc123...
  anthropic: sk-ant-api03-abc123...
  openai: sk-abc123...

You can add as many providers as you want. BujiCoder routes each model to its provider automatically based on the model ID prefix.

Using Environment Variables

API keys can also be set via environment variables. These are used as fallbacks when no key is in the config file:

# Add to your shell profile (~/.bashrc, ~/.zshrc, etc.)
export OPENROUTER_API_KEY="sk-or-v1-abc123..."
export ANTHROPIC_API_KEY="sk-ant-api03-abc123..."
export OPENAI_API_KEY="sk-abc123..."
Priority: Config file keys take precedence over environment variables.

Supported Providers

OpenRouter

Access 200+ models from multiple providers with a single API key. This is the recommended starting point.

Config keyopenrouter
Env variableOPENROUTER_API_KEY
Key formatsk-or-v1-...
Get a keyopenrouter.ai/keys

Model IDs use the OpenRouter format, e.g. anthropic/claude-sonnet-4, openai/gpt-4o, google/gemini-2.5-pro-preview.

How to Get Your OpenRouter API Key

  1. Go to openrouter.ai and create a free account (you can sign in with Google, GitHub, or email).
  2. Once logged in, navigate to openrouter.ai/keys (or click Keys in the dashboard sidebar).
  3. Click Create Key, give it a name (e.g. "bujicoder"), and copy the generated key. It starts with sk-or-v1-.
  4. Add credits to your account under Credits. Many models have free tiers, but paid models require a balance.
  5. Add the key to your config:
# ~/.bujicoder/bujicoder.yaml
api_keys:
  openrouter: sk-or-v1-your-key-here
Why OpenRouter? With one key you get access to models from OpenAI, Anthropic, Google, Meta, Mistral, xAI, and many others. BujiCoder's default model mappings all use OpenRouter model IDs, so it works out of the box with no extra configuration.

Ollama (Local Models)

Run models 100% locally on your own hardware. No API key needed — just provide the Ollama server URL.

Config keyollama_url
Defaulthttp://localhost:11434

Setup:

# 1. Install Ollama
curl -fsSL https://ollama.ai/install.sh | sh

# 2. Pull a model
ollama pull llama3

# 3. Add to config
api_keys:
  ollama_url: http://localhost:11434

modes:
  normal:
    main: ollama/llama3

Multiple Providers

You can use different providers for different agent roles. For example, use a cheap OpenRouter model for file exploration and a powerful Anthropic model for editing:

api_keys:
  openrouter: sk-or-v1-...
  anthropic: sk-ant-api03-...

modes:
  heavy:
    main: anthropic/claude-sonnet-4
    file_explorer: openai/gpt-4o-mini    # routed via OpenRouter
    sub_agent: anthropic/claude-sonnet-4
Security: Never commit your bujicoder.yaml file to version control. The config directory (~/.bujicoder/) is in your home folder by default, so this shouldn't happen accidentally.
