API Keys

BujiCoder talks directly to LLM providers using your API keys. No middleman, no markup.

Quick start: You only need one API key to get started. OpenRouter is the recommended default — it gives access to 200+ models with a single key, and BujiCoder's built-in model mappings are designed for it out of the box.
Important: Every model in your modes config must be available from a provider whose API key you have. If you only have an OpenRouter key, all models must use OpenRouter model IDs (e.g. anthropic/claude-sonnet-4, openai/gpt-4o). If you only have an Anthropic key, all models must be Anthropic models. See Configuration for per-provider sample configs.

Adding Keys to the Config

Add your API key under the api_keys section in ~/.bujicoder/bujicoder.yaml. You only need one:

# Minimal config — just one provider
api_keys:
  openrouter: sk-or-v1-abc123...

You can optionally add more providers. BujiCoder routes each model to its provider automatically based on the model ID prefix.

Using Environment Variables

API keys can also be set via environment variables. These are used as fallbacks when no key is in the config file:

# Add to your shell profile (~/.bashrc, ~/.zshrc, etc.)
export OPENROUTER_API_KEY="sk-or-v1-abc123..."
export ANTHROPIC_API_KEY="sk-ant-api03-abc123..."
export OPENAI_API_KEY="sk-abc123..."
Priority: Config file keys take precedence over environment variables.

Supported Providers

OpenRouter

Access 200+ models from multiple providers with a single API key. This is the recommended starting point.

Config keyopenrouter
Env variableOPENROUTER_API_KEY
Key formatsk-or-v1-...
Get a keyopenrouter.ai/keys

Model IDs use the OpenRouter format, e.g. anthropic/claude-sonnet-4, openai/gpt-4o, google/gemini-2.5-pro-preview.

How to Get Your OpenRouter API Key

  1. Go to openrouter.ai and create a free account (you can sign in with Google, GitHub, or email).
  2. Once logged in, navigate to openrouter.ai/keys (or click Keys in the dashboard sidebar).
  3. Click Create Key, give it a name (e.g. "bujicoder"), and copy the generated key. It starts with sk-or-v1-.
  4. Add credits to your account under Credits. Many models have free tiers, but paid models require a balance.
  5. Add the key to your config:
# ~/.bujicoder/bujicoder.yaml
api_keys:
  openrouter: sk-or-v1-your-key-here
Why OpenRouter? With one key you get access to models from OpenAI, Anthropic, Google, Meta, Mistral, xAI, and many others. BujiCoder's default model mappings all use OpenRouter model IDs, so it works out of the box with no extra configuration.

Ollama (Local Models)

Run models 100% locally on your own hardware. No API key needed — just provide the Ollama server URL.

Config keyollama_url
Defaulthttp://localhost:11434

Setup:

# 1. Install Ollama
curl -fsSL https://ollama.ai/install.sh | sh

# 2. Pull a model
ollama pull llama3

# 3. Add to config
api_keys:
  ollama_url: http://localhost:11434

modes:
  normal:
    main: ollama/llama3

One Key Is Enough

BujiCoder works with a single API key. The key determines which models you can use — every model in your modes config must be accessible through the provider(s) whose keys you've configured.

OpenRouter Only (Recommended)

The default model mappings are designed for OpenRouter. With just an OpenRouter key, everything works out of the box — no config changes needed:

api_keys:
  openrouter: sk-or-v1-your-key-here

# No modes section needed — defaults use OpenRouter models

Anthropic Only

If you only have an Anthropic key, set all models to Anthropic models:

api_keys:
  anthropic: sk-ant-api03-your-key-here

modes:
  normal:
    main: anthropic/claude-sonnet-4-20250514
    file_explorer: anthropic/claude-haiku-4-5-20251001
    sub_agent: anthropic/claude-sonnet-4-20250514
  heavy:
    main: anthropic/claude-sonnet-4-20250514
    file_explorer: anthropic/claude-haiku-4-5-20251001
    sub_agent: anthropic/claude-sonnet-4-20250514
  max:
    main: anthropic/claude-opus-4-20250514
    file_explorer: anthropic/claude-haiku-4-5-20251001
    sub_agent: anthropic/claude-sonnet-4-20250514

Multiple Providers

You can mix providers for different agent roles. When you have multiple keys, you can pick the best model from each provider:

api_keys:
  openrouter: sk-or-v1-...
  anthropic: sk-ant-api03-...

modes:
  heavy:
    main: anthropic/claude-sonnet-4          # direct Anthropic API
    file_explorer: openai/gpt-4o-mini        # routed via OpenRouter
    sub_agent: anthropic/claude-sonnet-4
Security: Never commit your bujicoder.yaml file to version control. The config directory (~/.bujicoder/) is in your home folder by default, so this shouldn't happen accidentally.
