Installation

BujiCoder ships as a single binary. No runtime dependencies required.

macOS / Linux (recommended)

Use the install script to download the latest release and place it in your PATH:

curl -fsSL https://community.bujicoder.com/install.sh | bash

This installs the buji binary to ~/.local/bin/. Make sure this directory is in your $PATH.

Build from Source

Requires Go 1.24+.

go install github.com/TechnoAllianceAE/bujicoder/cli/cmd/buji@latest

Or clone and build manually:

git clone https://github.com/TechnoAllianceAE/bujicoder.git
cd bujicoder
go build -o buji ./cli/cmd/buji
./buji

Windows

Download the latest .exe from GitHub Releases and add it to your PATH.

Verify Installation

buji --version

First Run

On first launch, BujiCoder runs a setup wizard that:

  1. Asks you to choose a setup mode (Quick or Advanced)
  2. Prompts for an API key from your chosen provider
  3. Creates the config file at ~/.bujicoder/bujicoder.yaml
You only need one API key. We recommend OpenRouter — it gives access to 200+ models with a single key, and BujiCoder's default model mappings are designed for it. Just paste your OpenRouter key and you're ready to code.

If you use a different provider (Anthropic, OpenAI, Groq, etc.), you'll also need to configure the modes section in your config to use models available from that provider. See API Keys for per-provider sample configs.

Quick Start with OpenRouter

  1. Sign up at openrouter.ai (free — use Google, GitHub, or email)
  2. Go to openrouter.ai/keys and click Create Key
  3. Add credits under Credits (many models have free tiers)
  4. Run buji and paste your key when prompted — that's it!

Local Models (Ollama / Llama.cpp)

BujiCoder works with local LLMs via Ollama or any llama.cpp server. In Advanced Setup, select Ollama or Llama.cpp and enter your server URL (or press Enter to use the default). BujiCoder will fetch available models from your server and let you assign them to agent roles.

# Start your local server first, then run:
buji
# Choose Advanced Setup → Ollama or Llama.cpp
# Press Enter to use the default URL (localhost:11434 / localhost:8080)
Tool support matters. BujiCoder's agents use tool/function calling to read files, edit code, run commands, and search your codebase. Not all local models support this. Recommended models with tool support:
  • llama3.1:8b / llama3.1:70b
  • qwen2.5-coder:7b / qwen2.5-coder:32b
  • mistral:7b
  • qwen3:8b
Models without tool support can still chat, but won't be able to use any agent tools.
Note: You can skip the wizard and create the config file manually. See Configuration.

Updating

BujiCoder checks for updates on launch (can be disabled). To update manually:

# Re-run the install script
curl -fsSL https://community.bujicoder.com/install.sh | bash

# Or from source
go install github.com/TechnoAllianceAE/bujicoder/cli/cmd/buji@latest

You can also use the /update command inside the TUI to check for updates.

Uninstalling

# Remove the binary
rm ~/.local/bin/buji

# Remove config and data (optional)
rm -rf ~/.bujicoder
